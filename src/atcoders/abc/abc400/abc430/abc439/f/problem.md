# F - Beautiful Kadomatsu

Problem: [AtCoder Beginner Contest 439 F](https://atcoder.jp/contests/abc439/tasks/abc439_f)

## Problem Statement

For a sequence `a = (a_1, a_2, ..., a_k)`, let:

- `x` be the number of indices `i` such that `2 <= i <= k - 1`, `a_{i-1} < a_i`, and
  `a_i > a_{i+1}`;
- `y` be the number of indices `i` such that `2 <= i <= k - 1`, `a_{i-1} > a_i`, and
  `a_i < a_{i+1}`.

The sequence is called **kadomatsu-like** if and only if `x > y`.

You are given a permutation `P` of `(1, 2, ..., N)`. Find the number of kadomatsu-like
subsequences of `P`. The subsequences do not have to be contiguous.

Print the answer modulo `998244353`.

## Constraints

- `1 <= N <= 3 * 10^5`
- `P` is a permutation of `(1, 2, ..., N)`.
- All input values are integers.

## Input

```text
N
P_1 P_2 ... P_N
```

## Output

Print the number of kadomatsu-like subsequences of `P`, modulo `998244353`.

## Sample 1

```text
Input
4
1 3 4 2

Output
4
```

The four kadomatsu-like subsequences are `(1, 3, 4, 2)`, `(1, 3, 2)`, `(1, 4, 2)`, and
`(3, 4, 2)`.

## Sample 2

```text
Input
1
1

Output
0
```

## Sample 3

```text
Input
20
11 10 18 13 12 16 5 19 7 6 17 4 9 1 14 2 20 15 8 3

Output
431610
```

## Ideas: Small-case DP

先写一个能够验证思路的小规模 DP。

只记录“处理到 `i` 为止”是不够的。把 `p[i]` 接到一个子序列后面时，会不会新增一个峰或谷，
取决于这个子序列最后一次是上升还是下降。因此，状态还需要记录子序列的结尾位置和最后一次比较的方向。

定义：

```text
dp[i][d][diff]
```

表示满足以下条件的子序列数量：

- 子序列以 `p[i]` 结尾；
- 子序列长度至少为 `2`；
- `diff = x - y`，其中 `x` 是峰的数量，`y` 是谷的数量；
- `d = 0` 表示最后一步下降，即前一个被选择的数大于 `p[i]`；
- `d = 1` 表示最后一步上升，即前一个被选择的数小于 `p[i]`。

### 初始化

枚举 `i1 < i`，由 `p[i1]` 和 `p[i]` 组成一个长度为 `2` 的子序列。

长度为 `2` 时还没有中间位置，所以没有峰或谷，`diff = 0`。比较 `p[i1]` 和 `p[i]`
即可确定最后一步的方向 `d`。

### 转移

枚举倒数第二个位置 `i1 < i`，把 `p[i]` 接到所有以 `p[i1]` 结尾的状态后面。
令新产生的比较方向为 `d1`：

- 如果 `p[i1] > p[i]`，则 `d1 = 0`，新的一步是下降；
- 如果 `p[i1] < p[i]`，则 `d1 = 1`，新的一步是上升。

然后比较旧方向 `d` 和新方向 `d1`：

- `d = d1`：方向没有改变，没有产生新的峰或谷，`diff` 不变；
- `d = 1, d1 = 0`：由上升变为下降，`p[i1]` 是一个峰，`diff` 加 `1`；
- `d = 0, d1 = 1`：由下降变为上升，`p[i1]` 是一个谷，`diff` 减 `1`。

每个非空 DP 状态都可以按照这个规则转移到 `dp[i][d1][newDiff]`。

### 统计答案

一个子序列是门松的条件是 `x > y`，也就是 `diff > 0`。因此，对每个结尾 `i`，
把 `dp[i][0][diff]` 和 `dp[i][1][diff]` 中所有 `diff > 0` 的状态加入答案。

### 复杂度

结尾位置 `i`、倒数第二个位置 `i1` 和 `diff` 都需要枚举，所以这个实现的时间复杂度是
`O(N^3)`，空间复杂度是 `O(N^2)`。它只适合验证小规模情况，还不能通过 `N <= 3 * 10^5`
的数据范围。

## Ideas: Remove the `diff` Dimension

进一步观察相邻元素之间的大小关系。用 `1` 表示上升，用 `0` 表示下降。例如：

```text
1, 4, 3, 5, 2
   1  0  1  0
```

一个峰对应 `1 -> 0`，一个谷对应 `0 -> 1`。这两种方向变化一定交替出现，因此
`diff = peaks - valleys` 不可能任意增大或减小，只可能是 `-1`、`0`、`1`：

```text
diff =  1：第一次比较是上升，最后一次比较是下降
diff = -1：第一次比较是下降，最后一次比较是上升
diff =  0：第一次和最后一次比较方向相同
```

所以，一个子序列是门松的充要条件是：

```text
第一次比较是上升，并且最后一次比较是下降。
```

不再需要记录完整的 `diff`。只考虑第一次比较为上升的子序列，因为第一次比较为下降的
子序列无论以后怎样变化，都不可能满足 `peaks - valleys = 1`。

### 两个半状态

按照最后一次比较的方向，把第一次比较为上升的子序列分成两个状态：

```text
seg[0][v]：以值 v 结尾，第一次比较上升，最后一次比较也上升
seg[1][v]：以值 v 结尾，第一次比较上升，最后一次比较下降
```

可以把 `seg[0]` 看作“半个峰”，把 `seg[1]` 看作“半个谷”。`seg[1]` 中的每个状态
已经满足第一次上升、最后一次下降，因此都是一个门松子序列。

按顺序处理 `v = p[i]`。设所有值已经减 `1`，线段树使用下标 `[0, N)`。

如果前一个结尾值小于 `v`，追加 `v` 后最后一步是上升，所以新状态进入 `seg[0]`：

```text
newHalfPeak = earlierLess(v)
            + query(seg[0], 0, v)
            + query(seg[1], 0, v)
```

其中 `earlierLess(v)` 是此前出现过且小于 `v` 的单个元素数量。每个这样的元素和当前的
`v` 组成一个长度为 `2`、第一次比较为上升的新子序列。

如果前一个结尾值大于 `v`，追加 `v` 后最后一步是下降，所以新状态进入 `seg[1]`：

```text
newHalfValley = query(seg[0], v, N)
              + query(seg[1], v, N)
```

这里必须查询两个状态：

- `seg[0] -> seg[1]` 表示由上升变成下降，闭合一个峰；
- `seg[1] -> seg[1]` 表示继续下降，子序列仍然满足第一次上升、最后一次下降。

同理，在小于 `v` 的一侧：

- `seg[0] -> seg[0]` 表示继续上升；
- `seg[1] -> seg[0]` 表示由下降变成上升，闭合一个谷。

当前新产生的所有 `newHalfValley` 都应加入答案。完成查询以后，再把 `newHalfPeak` 和
`newHalfValley` 更新到值 `v` 的位置。

查询大于当前值的范围时使用 `[v, N)` 是安全的：`P` 是一个排列，此前不会出现另一个
值为 `v` 的元素；并且当前 `v` 要在所有查询完成后才插入。因此，查询时位置 `v` 仍然为空，
`[v, N)` 与严格的大于 `v` 等价。

`earlierLess(v)` 可以用另一棵保存已出现单个元素数量的树维护。每轮最后在位置 `v` 加 `1`。

### 两个半状态还可以合并

追加当前值以后，新的最后方向完全由“旧结尾值和当前值的大小关系”决定，不依赖旧状态
最后是上升还是下降。因此，可以令：

```text
ways[v]：以值 v 结尾，并且第一次比较为上升的所有子序列数量
```

那么处理 `v` 时：

```text
answer += query(ways, v, N)

newWays = query(ways, 0, N) + earlierLess(v)

update(ways, v, newWays)
update(seen, v, 1)
```

第一行统计旧结尾大于 `v` 的状态：追加 `v` 后，它们的最后一步下降，所以形成新的门松
子序列。第二行表示所有已经以一次上升开头的子序列都可以继续追加 `v`，再加上以当前 `v`
作为第二个元素的新上升数对。

无论保留两个半状态还是合并为 `ways`，每个元素只进行常数次区间查询和单点更新，时间复杂度
都是 `O(N log N)`，空间复杂度是 `O(N)`。所有计数和加法都需要对 `998244353` 取模。

## Solution Summary

把子序列中每两个相邻元素的比较记为上升或下降。峰是“上升变下降”，谷是“下降变上升”。
峰和谷一定交替出现，所以一个子序列满足 `peaks > valleys`，当且仅当它的第一次比较上升、
最后一次比较下降。

代码用三棵树按“结尾的值”维护状态：

- `t0[v]`：以值 `v` 结尾，第一次比较上升，最后一次比较也上升的子序列数量；
- `t1[v]`：以值 `v` 结尾，第一次比较上升，最后一次比较下降的子序列数量；
- `t2[v]`：值 `v` 是否已经出现，用来统计可以和当前值组成第一个上升数对的单个元素。

按原顺序处理当前值 `v`。查询发生在插入 `v` 之前，而且 `P` 是排列，所以：

- `query(0, v)` 只包含结尾值小于 `v` 的旧状态；
- `query(v, N)` 只包含结尾值大于 `v` 的旧状态。

最后一步上升的新状态数为：

```text
tmp0 = less(t2) + less(t0) + less(t1)
```

第一项用一个更小的旧元素和 `v` 新建长度为 `2` 的上升子序列；后两项把已有子序列接上
一个更大的值 `v`。无论旧状态最后上升还是下降，追加后最后一步都变成上升。

最后一步下降的新状态数为：

```text
tmp1 = greater(t0) + greater(t1)
```

这些子序列的第一次比较都上升，追加 `v` 后最后一次比较下降，因此 `tmp1` 中的每个状态
都是新的门松子序列，需要全部加入答案。这里既要统计 `t0 -> t1`，也要统计继续下降的
`t1 -> t1`；忽略后者会漏掉像 `(1, 4, 3, 2)` 这样的子序列。

完成所有查询后，在位置 `v` 分别加入 `tmp0`、`tmp1`，并在 `t2` 中标记 `v` 已经出现。
这样可以保证每个子序列只在其最后一个元素被处理时生成一次。

### Correctness

根据峰谷交替的性质，门松子序列恰好是第一次比较上升、最后一次比较下降的子序列。
`t0` 和 `t1` 只保存第一次比较上升的子序列，并根据最后一次比较方向完整划分它们。

处理 `v` 时，每个旧状态能否转移、转移到 `t0` 还是 `t1`，完全由旧结尾值与 `v` 的大小
关系决定。`t2` 又恰好补充所有以 `v` 为第二个元素的新上升数对。因此，转移不会遗漏或
重复任何第一次比较上升的子序列。所有且仅有进入 `t1` 的新状态满足门松条件，所以累加
每轮的 `tmp1` 得到正确答案。

### Complexity

每个元素执行常数次树状数组的区间查询和单点更新：

- 时间复杂度：`O(N log N)`；
- 空间复杂度：`O(N)`。
