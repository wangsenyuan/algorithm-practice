# Problem

有 $n$ 个栈，每个栈恰好两张照片（顶部和底部）。Alice 和 Bonnie 轮流操作（Alice 先手），每次可以：
- 取走任意一个栈的**栈顶**照片（底部照片需要顶部被取走后才可取）；
- 或者 **跳过（pass）**。

当所有照片被取完，或者**连续两人都 pass** 时，游戏结束。

每张照片有两个非负整数值 $(a, b)$，取走照片的人获得对应的幸福值（Alice 获得 $a$，Bonnie 获得 $b$）。双方都以最优策略最大化 **自己的幸福值减去对方的幸福值**。

求最优博弈下 Alice 总幸福值减去 Bonnie 总幸福值。

**关键要素：**
- 栈约束：必须先取顶部才能取底部
- Pass 机制：玩家可以主动跳过，连续两次 pass 结束游戏（部分照片可能不被取走）
- 零和博弈：双方最大化 (己方 - 对方)
- 规模：$n \le 10^5$，值 $\le 10^9$

## Input

- The first line contains a single integer $n$ ($1 \le n \le 100\,000$) — the number of stacks.
- Each of the next $n$ lines contains four non-negative integers $a_1, b_1, a_2, b_2$ (each $\le 10^9$): $a_1, b_1$ describe the **top** photo, $a_2, b_2$ the **bottom** photo of that stack.

## Output

Output a single integer: Alice's total happiness minus Bonnie's total happiness in an optimally played game.

## Examples

### Example 1

**Input:**

```
2
12 3 4 7
1 15 9 1
```

**Output:**

```
1
```

### Example 2

**Input:**

```
2
5 4 8 8
4 12 14 0
```

**Output:**

```
4
```

### Example 3

**Input:**

```
1
0 10 0 10
```

**Output:**

```
-10
```

## editorial
对于两张在不同堆的照片（幸福值为 a1 b1 a2 b1）：
如果 A 拿第一张，B 拿第二张，那么 x-y = a1-b2。
如果 A 拿第二张，B 拿第一张，那么 x-y = a2-b1。
如果 a1-b2 > a2-b1，移项得 a1+b1 > a2+b2。
由此可知，A 和 B 会争夺 a+b 更大的照片。a+b 可以视作一张照片的「综合价值」。
在没有任何约束的情况下，最佳策略是把 a+b 从大到小排序，然后两人依次拿照片。

对于两张在同一堆的照片（上面的幸福值为 a1 b1，下面的幸福值为 a2 b1），分类讨论：

1. a1+b1 >= a2+b2 的情况。
由于两人本来就会按照 a+b 从大到小的顺序拿，所以这和无约束的情况是一样的。

2. a1+b1 < a2+b2 的情况。
如果 A 拿上面的，B 拿下面的，那么 x-y = a1-b2。
如果 B 拿上面的，A 拿下面的，那么 x-y = a2-b1。
由于 a1+b1 < a2+b2，所以 a1-b2 < a2-b1。
对于 A 来说，如果让 B 先拿，x-y 更大（相比 A 先拿）。
对于 B 来说，如果自己先拿了，x-y 更大（相比 A 先拿），这不是自己希望看到的。
所以两个人都不希望自己先拿。在这种情况下，x-y 不变，或者说对于 x-y 的收益是 0。
但如果 a1-b2 > 0，那么 A 最终还是会先拿上面的，毕竟这可以让 x-y 变大。
但如果 a2-b1 < 0，那么 B 最终还是会先拿上面的，毕竟这可以让 x-y 变小。
否则，两人都不会动这一堆的照片。
所以对于 a1+b1 < a2+b2 的情况，两人的策略是直接确定的，（这两张照片）无需参与排序。
