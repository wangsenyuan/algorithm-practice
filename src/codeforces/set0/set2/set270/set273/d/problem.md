# Problem

**题意简述**：n×m 网格，将若干格子涂黑。合法图案需满足：(1) 至少一个黑格；(2) 黑格四连通；(3) 任意两黑格间，仅沿黑格走的最短路径长度 = 曼哈顿距离。求合法图案数量 mod 10^9+7。

## Input

- The first line contains two integers n and m — the sizes of the piece of paper (1 ≤ n, m ≤ 150).

## Output

- In a single line print the remainder after dividing the answer by 1000000007 (10^9 + 7).

## Examples

### Example 1

**Input**

```text
2 2
```

**Output**

```text
13
```

### Example 2

**Input**

```text
3 4
```

**Output**

```text
571
```

### ideas
1. 如果有一行，最左边的black是l，最右边的是r, 那么 l...r中间都必须是黑色的
2. 且满足某一行i是最宽的一行，两头慢慢缩减，且被包含起来 (这样子，列也是符合同样的情形)
3. dp[i][l][r] 表示到第i行，且区间为l...r（且它上面的行，都被它包含）的方案数
4. dp[i][l][r] = sum over dp[i-1][l1][r1] where l <= l1 and r1 <= r
5. 先计算一个fp[l][r]表示，所有dp[i-1][l1][r1], where l <= l1 <= r1 <= r 的和
6. fp[l][r] = fp[l][r-1] + fp[l+1][r] - fp[l+1][r-1] + dp[l][r] 
7. 但是貌似会造成重复计数的情况，比如有两行（或者多行）是一样最长的情况

## Fix Algorithm

上面的“选一行作为最宽行”的想法是不对的，因为合法图形不一定存在一行包含所有其它行。比如区间序列 `[0,1] -> [1,2] -> [2,3]` 也是合法的，但不会被那个 DP 统计到。

### 1. 关键性质

把每个非空行表示成一个连续区间 `[l_i, r_i]`。

题目的条件等价于：

1. 每一行的黑格必须连续。
2. 非空行必须形成一个连续的行段。
3. 左端点序列 `l_i` 最多只会“拐弯”一次：
   先单调不增，再单调不减。
4. 右端点序列 `r_i` 最多只会“拐弯”一次：
   先单调不减，再单调不增。

也就是说，一个合法图形可以看成若干行区间叠出来的“L1 凸”图形。

### 2. DP 状态

设当前已经考虑了若干连续非空行，最后一行区间是 `[l, r]`。

再记录左右边界是否已经发生过“转向”：

- `00`：左边界还没转向，右边界还没转向
- `01`：左边界还没转向，右边界已经转向
- `10`：左边界已经转向，右边界还没转向
- `11`：左边界已经转向，右边界已经转向

记四个 DP 表：

- `s00[l][r]`
- `s01[l][r]`
- `s10[l][r]`
- `s11[l][r]`

表示高度固定时，以 `[l, r]` 结尾且处于对应转向状态的方案数。

初始时，高度为 1，任意 `l <= r` 都有一种方案，因此：

- `s00[l][r] = 1`

### 3. 状态转移

从上一行区间 `[l', r']` 转移到当前行 `[l, r]` 时：

- 如果左边界还没转向，那么允许 `l' >= l`
- 一旦出现 `l' < l`，说明左边界开始向右收缩，之后只能继续保持这种方向
- 右边界同理：
  - 没转向时允许 `r' <= r`
  - 一旦出现 `r' > r`，说明右边界开始向左收缩

于是四种状态分别对应四类矩形求和：

- `t00[l][r]`：上一状态必须也是 `00`，且 `l' >= l, r' <= r`
- `t01[l][r]`：右边界这一步发生转向，或之前已经转向
- `t10[l][r]`：左边界这一步发生转向，或之前已经转向
- `t11[l][r]`：左右边界都已经转向

这些转移本质上都是对某些 `(l', r')` 区域求和，因此可以对每个状态表预处理二维前缀和，把单次转移做到 `O(1)`。

### 4. 复杂度

- 状态数是 `O(m^2)`
- 一共做 `n` 层转移
- 每层利用二维前缀和完成转移

总复杂度：

- `O(n * m^2)`

空间复杂度：

- `O(m^2)`

### 5. 计数方式

先统计“恰好有 `h` 个连续非空行”的合法区间序列数量，再乘上这段连续非空行在整张 `n x m` 网格中的放置位置数 `n - h + 1`。

最后对所有 `h = 1..n` 求和即可。

### 6. Implementation Notes

#### What does `buildPrefix()` do?

`buildPrefix(a)` 会对一个二维状态表 `a[l][r]` 构造二维前缀和 `p`，满足：

- `p[i+1][j+1]` 等于子矩形 `a[0..i][0..j]` 的元素和

这样就可以通过一个 `rect()` 函数在 `O(1)` 时间内求任意矩形区域和，而不需要每次重新枚举所有 `(l', r')`。

它的作用就是加速四类状态转移：

- `s00[l][r]`
- `s01[l][r]`
- `s10[l][r]`
- `s11[l][r]`

因为这些转移本质上全是“对某个 `(l', r')` 范围批量求和”。

#### Why does the `h = 1` case not use `* n` later?

其实乘过了，只是单独提前处理了。

代码里先做了：

```go
oneRow := sumAll(s00)
for range n {
	ans = add(ans, oneRow)
}
```

这里：

- `s00[l][r] = 1` 表示“恰好只有 1 个非空行，且该行区间为 `[l, r]`”的方案
- `oneRow` 是所有单行图形的总数
- 单行图形可以放在整张网格的任意一行上，所以有 `n` 种放法

因此这段代码等价于：

```go
ans += oneRow * n
```

后面的主循环是从 `h = 2` 开始的，所以 `h = 1` 不会再重复计算一次。
