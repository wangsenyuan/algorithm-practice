# E. Interactive (Maximum Element)

This is an **interactive** problem.

While procrastinating at work, Dilhan the elf stumbled upon two arrays \(a\) and \(b\). Initially, both consist of a single integer \(2^k\) (i.e., \(a = b = [2^k]\)), where \(k\) is a non-negative integer.

Dilhan then applied the following two types of operations an arbitrary number of times (possibly zero), in any order:

1. **Flatten** — Choose either \(a\) or \(b\), and select any element \(x\) that is maximal within that array (\(x\) does not need to be maximal in the other array). Then, replace \(x\) with two copies of \(\frac{x}{2}\) in the same position. This operation can only be applied if \(x\) is even.
2. **Concatenate** — Set both \(a\) and \(b\) to be \(a + b\), where \(+\) denotes array concatenation.

After performing these operations, Dilhan discards \(b\), hides \(a\) from you, and challenges you to a game.

Let \(n\) be the length of the hidden array \(a\). You may make the following query:

- Choose an interval \([l, r]\) (\(1 \le l \le r \le n\)), and Dilhan will tell you the sum \(a_l + a_{l+1} + \cdots + a_r\).

**Task:** Determine the value of the maximum element of \(a\) by making at most **300** queries.

## Input

Each test contains multiple test cases. The first line contains the number of test cases \(t\) (\(1 \le t \le 100\)). The description of the test cases follows.

The first line of each test case contains a single integer \(n\) (\(1 \le n \le 10^5\)) — the length of \(a\).

It is guaranteed that \(1 \le a_i \le 2^{30}\), and the array \(a\) can be generated by the process described in the statement.

It is guaranteed that the sum of \(n\) over all test cases does not exceed \(10^5\).

## Interaction

For each test case, you are first given an integer \(n\) — the length of the hidden array \(a\). You may then make up to **300** queries.

**To make a query**, print a line in the following format:

- `? l r` (\(1 \le l \le r \le n\)) — the indices chosen for the query.

After you print a query, the interactor will respond with a single integer — the sum \(a_l + a_{l+1} + \ldots + a_r\).

**To report the maximum**, print your answer in the following format:

- `! m` (\(1 \le m \le 2^{30}\)) — the value of the maximum element.

Printing the answer does **not** count as one of the 300 queries.

After printing the answer, proceed to the next test case. If the current test case is the last one, terminate the program.

After printing each query, output a newline and **flush** the output. Otherwise you may get *Idleness limit exceeded*.

If, at any interaction step, you read \(-1\) instead of valid data, your solution must exit immediately (invalid query or other mistake). Failing to exit can yield an arbitrary verdict because your program would keep reading from a closed stream.

The interactor is **not adaptive**: the array \(a\) is fixed before any queries and does not change during the interaction.

Hacks are disabled in this problem.

**Flushing:** `fflush(stdout)` or `cout.flush()` in C++; `sys.stdout.flush()` in Python; see your language’s documentation for other environments.

## Note

In the first test case, Dilhan’s hidden array \(a\) is \([1,1,1,1,2,2,2,1,1,2,2]\). It can be generated by the following process:

| Step | Type                                            | \(a\) after operation       | \(b\) after operation       |
| ---: | :---------------------------------------------- | :-------------------------- | :-------------------------- |
|    0 |                                                 | \([4]\)                     | \([4]\)                     |
|    1 | Flatten \(a\): \([4] \to [2,2]\)                | \([2,2]\)                   | \([4]\)                     |
|    2 | Flatten \(b\): \([4] \to [2,2]\)                | \([2,2]\)                   | \([2,2]\)                   |
|    3 | Flatten \(a\): \([2,2] \to [2,1,1]\)            | \([2,1,1]\)                 | \([2,2]\)                   |
|    4 | Concatenate                                     | \([2,1,1,2,2]\)             | \([2,1,1,2,2]\)             |
|    5 | Flatten \(a\): \([2,\ldots] \to [1,1,1,1,2,2]\) | \([1,1,1,1,2,2]\)           | \([2,1,1,2,2]\)             |
|    6 | Concatenate                                     | \([1,1,1,1,2,2,2,1,1,2,2]\) | \([1,1,1,1,2,2,2,1,1,2,2]\) |

For the query `? 3 8`, the jury responded with \(1+1+2+2+2+1 = 9\).

For the query `? 1 4`, the jury responded with \(1+1+1+1 = 4\).

For the query `? 5 11`, the jury responded with \(2+2+2+1+1+2+2 = 12\).

The maximum element of \(a\) is \(2\).

In the second test case, \(a\) has only one element. The query `? 1 1` yields that value \(1\), which is also the maximum.

In the third test case, Dilhan’s hidden array \(a\) is \([4,4,4,4,4,4,4,4]\). It can be generated by:

| Step | Type        | \(a\) after operation | \(b\) after operation |
| ---: | :---------- | :-------------------- | :-------------------- |
|    0 |             | \([4]\)               | \([4]\)               |
|    1 | Concatenate | \([4,4]\)             | \([4,4]\)             |
|    2 | Concatenate | \([4,4,4,4]\)         | \([4,4,4,4]\)         |
|    3 | Concatenate | \([4,4,4,4,4,4,4,4]\) | \([4,4,4,4,4,4,4,4]\) |

The maximum element is \(4\).

In the fourth test case, Dilhan’s hidden array \(a\) is \([2^{29}, 2^{29}, 2^{30}, 2^{29}, 2^{28}, 2^{28}, 2^{29}, 2^{29}]\).
