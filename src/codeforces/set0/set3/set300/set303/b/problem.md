# Problem

给定一个大小为 $n \\times m$ 的矩形网格，坐标范围为 $0 \\le x \\le n$, $0 \\le y \\le m$。给定一个点 $(x, y)$，以及一个长宽比例 $(a, b)$。

要在网格内选取一个子矩形，其左下角为 $(x_1, y_1)$，右上角为 $(x_2, y_2)$，要求：

- **包含给定点**：$0 \\le x_1 \\le x \\le x_2 \\le n$，$0 \\le y_1 \\le y \\le y_2 \\le m$；
- **边与坐标轴平行，且坐标为整数**；
- **长宽比固定**：$(x_2 - x_1) : (y_2 - y_1) = a : b$。

在所有满足条件的子矩形中：

1. **优先**选择面积最大的（在比例固定时等价于边长的倍数最大）；
2. 如有多个面积最大的，选择其中心与点 $(x, y)$ 的欧氏距离最小的那个；
3. 若仍有多个，则在四元组 $(x_1, y_1, x_2, y_2)$ 的字典序中取最小的。

## Input

一行六个整数 $n, m, x, y, a, b$，含义如下：

- 网格大小：$1 \\le n, m \\le 10^9$；
- 给定点坐标：$0 \\le x \\le n$, $0 \\le y \\le m$；
- 比例参数：$1 \\le a \\le n$, $1 \\le b \\le m$。

## Output

输出四个整数 $x_1, y_1, x_2, y_2$，表示选出的子矩形左下角和右上角坐标。

## Examples

### Example 1

**Input**

```text
9 9 5 5 2 1
```

**Output**

```text
1 3 9 7
```

### Example 2

**Input**

```text
100 100 52 50 46 56
```

**Output**

```text
17 8 86 92
```

## Proof

当前实现使用的是直接构造，证明分成两部分：

1. 先证明最大面积的矩形尺寸唯一由比例决定。
2. 再证明固定尺寸后，按代码那样平移一定得到“最近且字典序最小”的答案。

### 1. 把比例化成最简

设：

- `g = gcd(a, b)`
- `a' = a / g`
- `b' = b / g`

任何满足 `(x2-x1):(y2-y1) = a:b` 的整数矩形，都必须写成：

- `x2 - x1 = a' * k`
- `y2 - y1 = b' * k`

其中 `k` 是正整数。

这是因为：

- `a'` 和 `b'` 互质。
- 若宽高之比是 `a':b'`，那么宽一定是 `a'` 的倍数，高一定是 `b'` 的倍数。
- 并且它们必须共享同一个缩放倍数 `k`。

所以问题已经转化成：

- 选一个尽量大的整数 `k`
- 让矩形大小为 `w = a' * k, h = b' * k`

### 2. 最大面积等价于最大 `k`

面积为：

- `w * h = (a' * k) * (b' * k) = a' * b' * k^2`

这里 `a' * b'` 是常数，所以面积随 `k` 单调递增。

因此：

- 面积最大 <=> `k` 最大

### 3. 最大可行缩放倍数

矩形必须放在 `n x m` 的大矩形内部，所以必须满足：

- `a' * k <= n`
- `b' * k <= m`

因此：

- `k <= n / a'`
- `k <= m / b'`

所以所有可行解都满足：

- `k <= min(n / a', m / b')`

另一方面，取

- `k = min(n / a', m / b')`

显然仍然满足两个约束，所以它本身可行。

故最大可行缩放倍数就是：

- `k = min(n / a', m / b')`

这一步证明了：  
代码先约分，再取这个 `k`，得到的 `(w, h)` 一定是面积最大的。

### 4. 固定宽高后，左右和上下可以独立处理

固定：

- `w = a' * k`
- `h = b' * k`

接下来只需选左下角 `(x1, y1)`。

包含点 `(x, y)` 的条件变成：

- `0 <= x1 <= x <= x1 + w <= n`
- `0 <= y1 <= y <= y1 + h <= m`

等价于：

- `max(0, x-w) <= x1 <= min(x, n-w)`
- `max(0, y-h) <= y1 <= min(y, m-h)`

所以：

- `x1` 的可选范围只和 `x, w, n` 有关
- `y1` 的可选范围只和 `y, h, m` 有关

而矩形中心到点 `(x, y)` 的距离平方是：

- `(x1 + w/2 - x)^2 + (y1 + h/2 - y)^2`

因为这两项彼此独立，所以：

- 分别最小化 `x` 方向和 `y` 方向的偏差
- 就等价于最小化整体欧氏距离

这就是为什么代码里可以写成一个一维函数 `place(...)`，分别处理横坐标和纵坐标。

### 5. 一维问题怎么最优放置

考虑一维版本：

- 总区间是 `[0, limit]`
- 线段长度是 `length`
- 必须覆盖点 `pos`

那么左端点 `left` 的可行范围是：

- `L = max(0, pos - length)`
- `R = min(pos, limit - length)`

线段中心是：

- `left + length/2`

目标是让中心尽量靠近 `pos`，也就是最小化：

- `|left + length/2 - pos|`

等价于最小化：

- `|2*left + length - 2*pos|`

如果 `left` 可以取实数，最优点显然是：

- `left = pos - length/2`

现在 `left` 必须是整数，所以最优整数点就是：

- 离 `pos - length/2` 最近的可行整数

代码写成：

```go
cand := (2*pos - length) / 2
```

然后把它夹到区间 `[L, R]` 中：

- 如果 `cand < L`，只能取 `L`
- 如果 `cand > R`，只能取 `R`
- 否则直接取 `cand`

因此 `place(...)` 返回的一定是一维最优位置。

### 6. 为什么还能保证字典序最小

若存在两个不同的整数左端点，都能达到相同的最小中心距离，那么它们一定是相邻整数，且较小的那个字典序更优。

代码在“正好居中落在两个整数中间”时，使用：

```go
cand := (2*pos - length) / 2
```

在这里相当于向下取整，所以会优先选更小的左端点。

于是：

- `x1` 会取到满足最小距离的最小值
- `y1` 也会取到满足最小距离的最小值

因此四元组：

- `(x1, y1, x1+w, y1+h)`

自然就是字典序最小的那个。

### 7. 结论

算法流程就是：

1. 用 `gcd` 把比例 `(a, b)` 约成最简。
2. 取最大缩放倍数 `k = min(n/a, m/b)`。
3. 得到最大面积矩形大小 `w = a*k, h = b*k`。
4. 分别在 `x` 和 `y` 方向上，把这个固定大小的矩形放到离 `(x, y)` 最近的位置。

因此该构造同时满足：

- 面积最大
- 中心距离最小
- 若仍并列则字典序最小

时间复杂度：

- `O(log min(a, b))`

其中只有 `gcd` 需要这个复杂度，其余步骤都是 `O(1)`。
